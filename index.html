Projects

Here is the list of the projects I have done through my time as a bachelor's student and afterwards. click each of the following for more; I have included the related courses and certificates in each.

I learned C++ programming language in two courses offered by ACECR(Academic Center for Education, Culture and Research) when I was 15 and like all other Electrical Engineering students in our university, I was again taught C++ in the "fundamentals of programming" course in my second term. The final Project of this course was "Bignum Calculator". This calculator was able to use the main operations for numbers up to 100 digits. You can check the code for this project here on my GitHub.

I learned Verilog in the "digital logic" course of the university in my third semester. The final optional project for this course was a controller of a "Traffic Light System" in an intersection. You can check the code for this project here on my GitHub. Our professor agreed for me to be the teaching assistant of this course in next semester.

On my fourth term at the university, I took the "Digital Systems 2" course. As the final optional project for this course, I chose to make an "instrument tuner". This device took the sound and the name of the tunable note in as the input and showed a related pattern as the output. To do this I used an Arduino Uno board. You can check the Arduino code for this project here on my GitHub.

On my fifth term at the university, I took the "Power Electronics" course for which my team and I made a "Half bridge inverter" as an optional project. To me, the most interesting part of this project was using CodeVisionAVR on Atmega8 to apply PWM and MPWM to the inverter.

For the "Machine Learning" course, for which I also gained the related Coursera certificate, I worked on a supervised regression model about "Apply chance calculator" to students. The programming language I used for this project was Python, which I initially learned on course offered by the Shahid Beheshti University computer engineering student association and I later took the python courses on Coursera.

For the "Digital Control Systems" course, I worked on a project about "controlling the output voltage of a wind turbine simulator" with a teammate. The goal was to make an automatic mode for a turbine simulator setup that was in the lab. We used Arduino to read the output voltage from a ZMPT101B sensor and send it to a PLC by serial communication. in order to control the output voltage and keep it in a safe small range, we designed a PID controller using NI LabVIEW.

For the "Fundamentals of mechatronics" course, we made an "Inverted Pendulum" and used an Arduino board to apply an LQR controller to it. Although I did not consider this project over yet; because the controller did not give the best result. Therefor I came back to working on this project during my seventh term again.

The summer after My Sixth term, I was an intern at a company in Iran, for which I worked on a research related to the image recognition of traffic signs. Therefore I also studied to gain Coursera certificates for Deep Learning. And to come to a better understanding of machine learning algorithms, I also gained Coursera certificates for mathematics for machine learning. Since Python programming language is used in many useful related applications, to make sure about my skills in this programming language, I also gained the Coursera certificates for Python.

In my seventh semester I came back to working on the "Inverted Pendulum" project, as a project for two courses, using two different approaches to control this system. This project turned out to be one of the most interesting projects I have worked on during my time at Shahid Beheshti University. The following diagrams show the two different control structures that where applied to the Inverted pendulum.

For the "Industrial Control" course, we implemented a PID controller on the pendulum. this controller worked quite well. Although the PID coefficients that we used were not exactly the same as the ones that worked in simulation, we had to change them a bit to get better result.

For the "Modern Control Systems" course we implemented a state feedback controller on the pendulum. The state feedback controller used a state observer to estimate the system's four state. In this case the controller worked as well, although the simulation worked a lot better.
